# 新機能提案: 位置情報 × グルーピング × つぶやき

**日付**: 2026-03-15
**ステータス**: 採用決定（未実装）

---

## 背景・課題

- 現状は**投稿（しっかりした情報共有）**がアプリの中心
- しかし投稿頻度はそこまで高くならない見込み
- 「見るだけの人」が大半になり、DAUが伸びないリスク
- **投稿以外のアクション導線**が必要

---

## コンセプト：「半径○mの井戸端会議」

投稿＝残る情報共有、つぶやき＝流れる気軽な会話。
位置情報で**「今ここにいる人同士」**を自動的につなげることで、他アプリにない体験を作る。

```
┌─────────────────────────────────┐
│  📍 駅前エリア（半径300m）       │
│  👥 12人がこの近くにいます       │
│                                 │
│  🗨 「駅前のパン屋今日行列すごい」│
│  🗨 「公園の桜もう咲いてる！」    │
│  🗨 「この辺でランチおすすめある？」│
│      └→ 💬 3件の返信            │
│                                 │
│  [ つぶやく... ]                │
└─────────────────────────────────┘
```

---

## 3要素の相乗効果

| 要素 | 単体だと | 組み合わせると |
|---|---|---|
| 位置情報 | 地図で投稿見るだけ | **「同じ場所にいる」共感が生まれる** |
| グルーピング | 作るのが面倒、過疎る | **位置で自動グループ化 → 参加ハードル0** |
| つぶやき | Twitterの劣化版 | **ローカル限定で「ここだけの話」感** |

---

## グループの2種類

### A. 自動グループ（位置ベース）← 優先実装

- 今いる場所の半径で自動的につながる
- 「駅前」「○○公園周辺」「商店街」など
- **離れたら自然にフェードアウト → 気楽**
- 参加操作不要。位置にいるだけでつながる

### B. 固定グループ（コミュニティベース）← 後から追加

- マンション、町内会、学校区など
- 参加制。つぶやきはグループ内だけに見える
- 既存の投稿機能もグループ内に流せる

---

## フェードアウトの仕組み

距離と時間でつぶやきの表示がグラデーションする。

```
近い＆新しい  →  くっきり表示（不透明度100%）
近い＆古い    →  少し薄い（時間で自然に消える）
遠い＆新しい  →  薄く見える（離れたけどまだ残る）
遠い＆古い    →  消える
```

### 生活動線がフィードになる

- 朝の通勤で駅前のつぶやきを見る → 会社に着いたら消えてる
- 帰宅したらまた地元のつぶやきが見える
- 週末に公園行ったら公園周辺の話題が出てくる

### フェード表示イメージ

```
┌─────────────────────────────┐
│  📍 今いるエリア              │
│                             │
│  「このカフェ空いてて穴場」   │  ← くっきり（近い＆新しい）
│  「信号長すぎない？笑」      │  ← くっきり
│                             │
│  ─ さっきいた場所 ─          │
│                             │
│  「駅前の工事いつ終わるの」   │  ← 薄い（離れた）
│  「新しいラーメン屋できてる」 │  ← さらに薄い
│                             │
└─────────────────────────────┘
```

### 消え方のパターン

| パターン | 特徴 | 向いてる場面 |
|---|---|---|
| **時間消滅型** | 24h後に一律消える | シンプル。MVP向き |
| **距離フェード型** | 離れるほど薄く | 「今ここ」感が最も強い |
| **ハイブリッド型** | 距離+時間の両方 | 理想形。ただし複雑 |
| **反応延命型** | リアクションが多いと長く残る | 良い情報が自然に生き残る |

---

## MVP（最小構成）

最初は**時間消滅型（24h）+ 位置フィルター**で開始。

- つぶやきは24hで消える
- 自分の半径○m以内のつぶやきだけ表示
- 視覚的なフェード（不透明度）は後から追加
- **距離グラデーションは反応を見てから実装**

## 既存タブとの関係

| タブ | 現在の役割 | つぶやき機能との関係 |
|---|---|---|
| Pulse | AI検索 | **つぶやきフィードに変更が有力**（「地域の脈拍」に合う） |
| Nearby | 投稿のマップ表示 | 地図上につぶやきも浮かべる案あり（情報過多に注意） |
| Timeline | セクション別投稿一覧 | 変更なし。投稿（残る情報）はここ |
| Profile | マイページ | 変更なし |

---

## 競合との差別化

| サービス | NaviOsの優位性 |
|---|---|
| LINE | グループは招待制。「たまたま近くにいる人」とは繋がれない |
| Twitter/X | 全世界公開。ローカルな話題が埋もれる |
| NextDoor | 海外サービス。日本の地域文化に合わない |

NaviOsは**「位置で自動的に繋がる気軽さ」+「地域コミュニティの安心感」**を両立。

---

## 懸念点と対策

| 懸念 | 対策 |
|---|---|
| 過疎エリアで人がいない | 固定グループ（B）と併用。つぶやきに時間的残存期間（24h）を設けて「さっきの話題」も見える |
| プライバシー | 正確な位置は非公開。エリア名だけ表示。既存認証で実名性を担保 |
| 投稿との棲み分け | 投稿＝残る情報、つぶやき＝流れる会話。UIも明確に分ける |
| つぶやきにカテゴリは？ | 不要。カテゴリなしでさらに気軽に |
