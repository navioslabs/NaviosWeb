### 2. 終了済み投稿の視覚的区別

`isEnded: true` の投稿に対し、全コンポーネントで視覚フィードバックを追加。

| コンポーネント | 対応内容 |
|---|---|
| `PostCard.tsx` | `opacity: 0.6` + 「終了」バッジ |
| `PostListItem.tsx` | `opacity: 0.6` + タイトル横に「終了」バッジ |
| `app/post/[id].tsx` | 「この投稿は終了しました」バナー + アクションボタン無効化（テキスト「終了済み」） |
| `profile.tsx` | メタ行に「終了」ラベル表示 |

### 3. エラー・警告表示のスタイル統一

各画面で個別にハードコードされていたエラー・警告UIのカラーを `dangerBg` / `warningBg` 等の共通定数に統一。将来のテーマ変更時は `colors.ts` の1箇所を変更するだけで全画面に反映。

### 4. デザイントークンの導入開始

新規追加スタイルでは `constants/design.ts` のトークン（`FontSize.xs`, `FontWeight.bold`, `Spacing.xs`, `Radius.sm`）を使用。既存コードの全面置換はレイアウト崩れリスクを考慮しスコープ外。

マイページのアカウントセクションからメールアドレスを変更可能に。

- メールアドレス欄に鉛筆アイコンを追加（編集可能であることを示す）

- `supabase.auth.updateUser()` でAuth側のメールを更新 + `users` テーブルも同期
- 更新後に確認メール送信の案内をアラート表示

## 技術的な判断

### ネイティブモジュール回避
- `expo-linear-gradient` は開発ビルドの再ビルドが必要なため不採用
- 代わりにRN標準の `View` 重ね合わせ + `rgba()` でグラデーション・グラスモーフィズムを再現

### 変更されていない設計方針
- `rgba()` によるインライン透明度指定は従来通り許可
- `#fff` / `#000` はユニバーサルな値のため一部インライン使用を維持
- `app/post/create.tsx` の分割はP2タスクとして継続管理

### TypeScript検証
全変更ファイルの型チェックを実施し、エラーがないことを確認済みです。
